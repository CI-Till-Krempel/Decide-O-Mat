# Announcing Decide-O-Mat v1.6.3: Dependency Maintenance & Security Hardening

We are releasing **Decide-O-Mat v1.6.3** — a focused maintenance patch that keeps the dependency tree up to date, resolves several security audit findings, and tightens GDPR compliance for user account deletion.

## What's Fixed?

### GDPR: Argument Authorship Anonymized on Account Deletion
When a user deletes their account, all argument author fields within their decisions are now properly anonymized. Previously, deleted accounts could leave behind identifiable author data in the Firestore subcollections. Participant records are also cleaned up more efficiently in a single batch operation.

### Security Hardening (Firestore Rules & Backend)
A pre-launch audit identified and resolved critical issues including a Firestore rules bypass, authentication gaps in Cloud Functions, and residual PII exposure paths. All known gaps are now closed.

### npm Audit Vulnerabilities Resolved
All outstanding npm audit vulnerabilities in both the frontend and Cloud Functions packages have been resolved.

### Gemini CI Workflows
The `GEMINI_CLI_TRUST_WORKSPACE` environment variable is now correctly set across all Gemini CLI workflow steps, preventing spurious trust-boundary warnings in CI runs.
